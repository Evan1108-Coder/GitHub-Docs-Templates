# Starter Packs

Starter packs are copy-ready documentation sets for common project types.

Use these when you do not want to choose every file one by one.

## Packs

- `personal-project/` - best for first repos, homework, experiments, and small projects.
- `open-source-library/` - best for reusable packages that welcome contributors.
- `web-app/` - best for websites and apps with setup and support needs.
- `cli-tool/` - best for command-line tools.
- `docs-site/` - best for documentation websites.

## How To Use

1. Pick one starter pack.
2. Copy its files into your own repository.
3. Replace placeholders like `[PROJECT_NAME]`.
4. Delete sections that do not apply.

## Important

Some starter packs include hidden `.github` folders. If you copy files manually, make sure those hidden folders are copied too.
