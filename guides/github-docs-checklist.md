# GitHub Docs Checklist

Use this checklist before publishing a repository.

## Required For Most Projects

- [ ] `README.md` explains what the project does.
- [ ] Installation or setup steps are clear.
- [ ] Usage examples are included.
- [ ] A license is included if the project is public.
- [ ] Placeholders have been replaced.

## Helpful For Collaboration

- [ ] `CONTRIBUTING.md` explains how to help.
- [ ] Issue templates guide bug reports and feature requests.
- [ ] Pull request template guides changes.
- [ ] Code of conduct sets community expectations.
- [ ] Security policy explains private reporting.

## Helpful For Releases

- [ ] `CHANGELOG.md` exists.
- [ ] Release notes template exists.
- [ ] Versioning approach is explained.
