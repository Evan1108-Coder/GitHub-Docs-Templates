# Contributing To The Docs

Thanks for improving [PROJECT_NAME] documentation.

## What Helps

- Fix unclear explanations.
- Add examples.
- Improve grammar.
- Update outdated steps.
- Add screenshots only when they make the docs clearer.

## Writing Style

- Use plain language.
- Prefer short sections.
- Explain beginner terms.
- Include commands in code blocks.

## Pull Request Checklist

- [ ] Links work.
- [ ] Commands are current.
- [ ] Screenshots are up to date if included.
