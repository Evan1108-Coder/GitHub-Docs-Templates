# README Templates

A `README.md` is usually the first file people see in a GitHub repository. It should quickly explain what the project is, why it exists, how to use it, and where to go next.

## How To Use These Templates

Copy one file from this folder into your own repository, then rename it to:

```text
README.md
```

For example:

```text
README_minimal.md -> README.md
README_library.md -> README.md
```

Replace placeholders such as `[PROJECT_NAME]`, `[PROJECT_DESCRIPTION]`, and `[INSTALL_COMMAND]`.

## Template Choices

- `README_minimal.md` - small project or first repository.
- `README_open_source.md` - public project that welcomes contributors.
- `README_web_app.md` - website or web application.
- `README_library.md` - reusable package or library.
- `README_cli_tool.md` - command-line tool.
- `README_docs_site.md` - documentation website.
- `README_personal_project.md` - portfolio, school, or learning project.
